# 基础设施模块 (`infra/`)

## 职责
系统级配置管理：系统设置、通知渠道、审核日志、备份、导入导出、审批流、工具箱。

## 内部结构
```
infra/
├── routes/     # 16 路由文件（最大路由模块）
├── services/   # 18 服务文件
│   ├── backupService.ts      ← 备份服务（876 行）
│   └── ...
```

## 依赖关系
- 被全项目其他模块依赖（共享基础设施）
- 不依赖具体业务模块

## 关键说明
- 路由最多（16 个）、服务最多（18 个）— 因为涵盖太多不同领域
- 考虑拆分子域：`infra/notification/`, `infra/backup/`, `infra/approval/`, `infra/config/`
